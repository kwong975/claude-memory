# Contributing to claude-memory

Thanks for your interest in improving claude-memory.

## How to contribute

### Reporting issues

- Use [GitHub Issues](../../issues) for bugs and problems
- Use [GitHub Discussions](../../discussions) for questions and ideas
- Check existing issues before creating a new one

### Suggesting changes

1. Fork the repo
2. Create a branch: `feat/<description>` or `fix/<description>`
3. Make your changes
4. Open a pull request with a clear description of what and why

### What we're looking for

- Improvements to templates (better defaults, clearer structure)
- New examples showing different workflows
- Documentation improvements
- Integration guides for different Claude Code setups
- Bug fixes in the setup script

### Commit conventions

We use [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` — new feature or template
- `fix:` — bug fix
- `docs:` — documentation change
- `chore:` — maintenance (CI, deps, etc.)

### What doesn't fit

- Features that add dependencies (this project is intentionally zero-dependency)
- Changes that make the system more complex without clear benefit
- Opinionated workflow changes (the system is meant to be customizable)

## Code of Conduct

This project follows the [Contributor Covenant](CODE_OF_CONDUCT.md). Be kind.
